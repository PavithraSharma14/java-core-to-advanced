# 13. Object-Oriented Programming (OOP)

## 1. Overview

Object-Oriented Programming (OOP) is a programming paradigm that models real-world entities using objects and classes.

Until this point in the repository, we focused on procedural programming concepts such as loops, arrays, functions, and bit manipulation.

In this module, we shift from writing logic-focused programs to designing structured, scalable, and maintainable systems.

OOP enables:
- Code reusability
- Modularity
- Abstraction
- Encapsulation of data
- Real-world modeling

This is where programming starts becoming software engineering.

---

## 2. Why OOP Matters

In real-world systems:
- Banking systems manage Accounts
- E-commerce systems manage Products, Orders, Users
- Operating systems manage Processes and Threads

These are modeled as **objects**.

OOP helps:
- Organize large codebases
- Reduce duplication
- Improve maintainability
- Enable team collaboration

Most modern enterprise applications are built using OOP principles.

## 5. From Procedural to Object-Oriented

Procedural Programming:
- Focus on functions
- Data is separate
- Global state is common
- Harder to scale

Object-Oriented Programming:
- Focus on objects
- Data + behavior bundled together
- Controlled access
- Better scalability

This shift is critical for building production systems.

---

## 6. JVM Perspective

When working with OOP in Java:

- Class metadata is stored in the Method Area.
- Objects are allocated in the Heap.
- References are stored in the Stack.
- Constructors initialize object state.
- Method calls use dynamic dispatch in case of polymorphism.

Understanding memory behavior is essential for mastering OOP in Java.
